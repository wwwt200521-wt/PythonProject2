## Skill Rules (通知规则)

### 基本要求
1. **部门前缀**: 通知标题前必须加入部门名称，格式为 "xx部通知"
2. **默认部门**: 用户未提供部门时，用 'xx' 作为部门名（即标题为“xx部通知”，署名为“xx部”）
3. **必含要素**: 标题、正文、署名（部门名称）、日期（YYYY年M月D日 格式）
4. **多部门联合**: 格式为 "部门A、部门B、部门C联合通知"
5. **特殊标记**: 
   - 紧急通知：[紧急] 前缀加在标题前
   - 重要通知：[重要] 前缀加在标题前
   - 变更通知：[变更] 前缀加在标题前

### 内容规范
- 正文字数建议：100-500字
- 用语：简洁、专业、尊重
- 如有具体截止日期或操作要求，必须在正文中明确标注

---

## Output Format Template (标准输出模板)

```
[标记] xx部通知

【正文】
各位同仁：

[内容段落1]

[内容段落2 - 如需要]

感谢配合！

<div align="right"><strong>xx部</strong><br>YYYY年M月D日</div>
```

## Examples (案例)

### 案例 1：用户未提供部门 - 使用默认值
**用户输入**: 
```
请帮我写一个系统维护通知
```

**输出**:
```
xx部通知

各位同仁：

系统将于2026年4月25日 22:00-23:00 进行定期维护，届时系统将无法访问。请提前保存工作数据，造成的不便敬请谅解。

感谢配合！

<div align="right"><strong>xx部</strong><br>2026年4月23日</div>
```

### 案例 2：用户提供具体部门
**用户输入**:
```
技术部需要发一个关于代码审查流程更新的通知
```

**输出**:
```
技术部通知

各位同仁：

为了提高代码质量，技术部对代码审查流程进行了优化。请所有开发人员从2026年5月1日起按新流程进行Pull Request提交和审查。

新流程详情已发布至内部wiki，如有疑问，欢迎联系技术部。

感谢配合！

<div align="right"><strong>技术部</strong><br>2026年4月23日</div>
```

### 案例 3：紧急通知 + 多部门
**用户输入**:
```
财务部和法务部联合发布一个紧急通知，关于财务报表提交截止日期变更
```

**输出**:
```
[紧急] 财务部、法务部联合通知

各位同仁：

受年度审计时间调整影响，财务报表提交截止日期已变更。

**原截止日期**: 2026年5月15日
**新截止日期**: 2026年4月30日

请各部门负责人务必按新截止日期提交。如有困难，请立即与财务部沟通。

感谢配合！

<div align="right"><strong>财务部、法务部</strong><br>2026年4月23日</div>
```

### 案例 4：修改现有通知
**用户输入**:
```
帮我润色这个通知，部门是人力资源部：
原文："员工们要交一份表格"
```

**输出**:
```
人力资源部通知

各位同仁：

为了做好年度绩效评估工作，请各位员工于2026年4月30日前提交《年度工作总结表》。

表格模板已发送至各部门邮箱，若有疑问，欢迎咨询人力资源部。

感谢配合！

<div align="right"><strong>人力资源部</strong><br>2026年4月23日</div>
```

---

## Notes (注意事项)

- 确保所有日期格式一致（YYYY年M月D日）
- 避免冗长复杂的句子，优先使用要点形式
- 如涉及截止日期或强制要求，用**加粗**标注
- 多部门通知需确认各部门同意后再发送
